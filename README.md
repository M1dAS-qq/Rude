# Rude
RUDE (Robust Universal Data Encryptor) v42

Описание

RUDE (Robust Universal Data Encryptor) — это мощная программа для шифрования и дешифрования текстовых данных, разработанная M1dAS_qq. Программа обеспечивает безопасное преобразование текста с использованием современных криптографических алгоритмов и дополнительных функций оптимизации.

Основные возможности

· Многопоточное шифрование с поддержкой различных алгоритмов (AES-256-CTR, AES-256-GCM, AES-256-CBC)
· Адаптивное сжатие данных с выбором уровня (отсутствует, быстрое, оптимальное, максимальное)
· Интеллектуальная система алфавитов с автоматическим определением языка текста
· Контроль целостности данных через проверку контрольных сумм CRC32
· Гибкая система настроек для тонкой настройки процесса шифрования
· Поддержка двух языков интерфейса (русский и английский)
· Режим скрытия ключей для дополнительной безопасности

Руководство пользователя

Запуск программы

После запуска программы отображается баннер с версией и приглашение к вводу команд. Основной режим работы — интерактивный, с пошаговым вводом данных.

Основные команды и аргументы

Все аргументы команд указываются в квадратных скобках. Например: %mode1 [Ваш текст] или %mode2 [шифр] [ключ]

Команда %mode1 - шифрование текста

Назначение: Шифрование текстовых данных с генерацией ключа

Синтаксис:

· %mode1 - программа запросит текст для шифрования
· %mode1 [текст] - непосредственная передача текста для шифрования

Аргументы:

· [текст] - текст для шифрования (максимум 4000 символов)

Примеры:

```
%mode1 [Это секретное сообщение]
%mode1 [Hello world! This is confidential information.]
```

Команда %mode2 - дешифрование текста

Назначение: Дешифрование ранее зашифрованных данных

Синтаксис:

· %mode2 - программа запросит шифр и ключ по отдельности
· %mode2 [шифр] [ключ] - передача обоих параметров сразу

Аргументы:

· [шифр] - зашифрованные данные в формате base85 или hex
· [ключ] - ключ дешифрования в шестнадцатеричном формате (32 символа)

Примеры:

```
%mode2 [FXiZ>GAHQ+AZc+E_#@T] [A1B2C3D4E5F6789012345678901234567]
```

Команда %settings - управление настройками

Назначение: Просмотр и изменение параметров программы

Синтаксис:

· %settings - отображение текущих настроек и вход в интерактивный режим
· %settings [параметры] - применение настроек без входа в интерактивный режим

Аргументы:
Параметры задаются в форматеномер%значение, разделяются пробелами:

· 1%on - включение режима отладки
· 2%off - выключение скрытия ключа
· 3%en - установка английского языка интерфейса
· 4%random - случайные контрольные флаги
· 5%8 - установка 8 потоков обработки
· 6%gcm - выбор алгоритма AES-GCM
· 7%optimal - установка оптимального сжатия
· 8%auto - автоматический выбор алфавита

Примеры:

```
%settings [1%on] [3%en] [6%gcm]
%settings [2%off] [5%4] [7%fast]
```

Команда %help - справочная система

Назначение: Получение справки по командам программы

Синтаксис:

· %help - общая справка по всем командам
· %help [команда] - подробная справка по конкретной команде

Аргументы:

· [команда] - имя команды без знака % (mode1, mode2, settings, help, exit)

Примеры:

```
%help [mode1]
%help [settings]
```

Команда %exit - завершение работы

Назначение: Корректное завершение работы программы

Синтаксис: %exit

Детальное описание настроек

Программа поддерживает 8 основных настроек, которые можно изменять через команду %settings:

1. Режим отладки (Debug Mode)

Управляет выводом отладочной информации о процессе шифрования/дешифрования.

· on/1 - включить подробный вывод
· off/0 - выключить отладочную информацию

2. Скрытие ключа (Hide Key Mode)

Альтернативный формат вывода для повышенной безопасности.

· on/1 - ключ маскируется случайными данными
· off/0 - стандартный формат вывода ключа

3. Язык интерфейса (Language)

Выбор языка интерфейса программы.

· ru - русский язык
· en - английский язык

4. Контрольные флаги (Control Flags)

Идентификатор для генерации ключа шифрования.

· random - случайное значение (рекомендуется)
· hex_значение - заданное значение (1-5 hex цифр)

5. Количество потоков (Threads)

Оптимизация производительности для многопоточных систем.

· 1-16 - количество потоков обработки

6. Алгоритм шифрования (Algorithm)

Выбор криптографического алгоритма.

· ctr/0 - AES-256-CTR (рекомендуемый)
· gcm/1 - AES-256-GCM
· cbc/2 - AES-256-CBC

7. Уровень сжатия (Compression)

Степень сжатия данных перед шифрованием.

· none/0 - без сжатия
· fast/1 - быстрое сжатие
· optimal/2 - оптимальное сжатие (рекомендуется)
· max/3 - максимальное сжатие

8. Режим алфавита (Alphabet Mode)

Выбор системы символов для кодирования.

· auto/0 - автоматическое определение (рекомендуется)
· universal/1 - универсальный алфавит
· russian/2 - оптимизирован для русского текста
· english/3 - оптимизирован для английского текста

Процесс шифрования

1. Введите команду %mode1 или %mode1 [текст]
2. Если текст не указан, программа запросит его ввод
3. Программа автоматически определит язык текста и выберет оптимальный алфавит
4. После обработки будут показаны:
   · Зашифрованные данные (шифр)
   · Ключ дешифрования
   · Детальная информация о параметрах шифрования

Процесс дешифрования

1. Введите команду %mode2 или %mode2 [шифр] [ключ]
2. Если параметры не указаны, программа запросит их по отдельности
3. Программа выполнит проверку целостности данных и отобразит:
   · Расшифрованный текст
   · Информацию о параметрах шифрования
   · Результаты проверки контрольной суммы

Особенности работы

Программа автоматически обрабатывает специальные символы и символы из различных языков, заменяя отсутствующие в текущем алфавите символы на ближайшие аналоги. Это обеспечивает корректную работу с текстами разного происхождения.

Система контроля целостности данных предупреждает о возможных ошибках при дешифровании и позволяет обнаружить повреждение данных.

Рекомендации по использованию

· Сохраняйте ключ дешифрования в безопасном месте — без него восстановление данных невозможно
· При дешифровании обращайте внимание на предупреждения о несовпадении контрольных сумм
· Используйте режим отладки для лучшего понимания процессов шифрования/дешифрования
· Для текстов на конкретном языке устанавливайте соответствующий режим алфавита для улучшения сжатия
· Для большинства случаев используйте настройки по умолчанию - они обеспечивают оптимальный баланс безопасности и производительности

Технические особенности

Программа использует современные криптографические библиотеки и реализует проверенные методы безопасности. Поддержка различных алгоритмов шифрования позволяет выбрать оптимальный баланс между производительностью и безопасностью.
